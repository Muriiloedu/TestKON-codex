# Backlog inicial (histórias de usuário)

## Épico 1 — Registro e tratativa de manifestações

1. Como **morador**, quero abrir uma manifestação por categoria para registrar meu problema.
2. Como **morador**, quero anexar foto/vídeo para facilitar a análise.
3. Como **morador**, quero acompanhar o status em tempo real para saber quando serei atendido.
4. Como **administradora**, quero classificar e atribuir manifestações para responsáveis.
5. Como **síndico**, quero aprovar tratativas sensíveis para manter governança.

## Épico 2 — Indicadores e governança

1. Como **síndico**, quero ver os principais indicadores por período para orientar decisões.
2. Como **administradora**, quero monitorar SLAs para reduzir atrasos.
3. Como **gestão**, quero visualizar recorrência por local em um mapa de calor.

## Épico 3 — Reserva de áreas comuns

1. Como **morador**, quero reservar áreas comuns pela agenda digital.
2. Como **administradora**, quero configurar regras de reserva por espaço.
3. Como **síndico**, quero bloquear datas de manutenção/eventos internos.

## Épico 4 — Feira de trocas

1. Como **morador**, quero publicar itens para troca/doação/venda interna.
2. Como **morador**, quero conversar com interessados via chat.
3. Como **administradora**, quero moderar anúncios inadequados.

## Critérios de pronto (DoD)

- Requisitos funcionais implementados e testados.
- Controle de acesso validado por perfil.
- Registro de auditoria ativo para operações administrativas.
- Métricas mínimas expostas em dashboard.
- Documentação atualizada.
